How the score is built

Data sources

  • News — GDELT DOC 2.0 API, free, queried per corridor keyword every pipeline run.
  • AIS — aisstream.io, free tier, burst-sampled ~90 seconds per corridor per run (see limitations).
  • Prices — oilpriceapi.com, free tier, Brent and WTI.
  • Sanctions — OFAC's public SDN list, diffed against the previous snapshot each run.

Extraction

Each news article is passed to an LLM (Groq free tier) with a fixed extraction schema: corridor, actors, event_type, severity, confidence, summary. Before scoring, the 3–5 most similar past events for that corridor are retrieved from a local vector index (sentence-transformers embeddings, cosine similarity — no managed vector database) and given to the LLM as calibration anchors, so severity scoring stays consistent over time instead of drifting article to article. Articles with confidence below 0.4 are dropped rather than written into the graph.

Scoring formula

score = normalize(w1·news_pressure + w2·ais_pressure + w3·price_momentum + w4·sanctions_pressure)

  • news_pressure — each event's severity × confidence, decayed exponentially with a 60-hour half-life, summed across all events for that corridor. This is what makes the score behave like a live instrument: it cools down on its own if nothing new happens, rather than staying pinned at a stale high value.
  • ais_pressure — count of flagged anomalies (speed jumps, AIS transmission gaps) in the corridor's bounding box during the most recent sampling window.
  • price_momentum — a clipped z-score of the latest price against its recent rolling window; only upside momentum raises the score.
  • sanctions_pressure — new OFAC designations that name an entity on the corridor's watchlist (e.g. IRGC-linked shipping entities for Hormuz).

Limitations — stated deliberately

AIS is burst-sampled, not continuous. This is a local process without an always-on server, so each pipeline run opens a websocket for ~90 seconds per corridor rather than streaming continuously between runs. Anomaly detection quality scales with sample duration; a longer-running deployment would hold the connection open persistently instead.
The knowledge graph is JSON, not a graph database. At the data volumes this produces (hundreds to low thousands of events), a committed JSON file is simpler and free. The upgrade path if corridor/entity relationships grow complex enough to need real graph traversal is Neo4j or a similar store, using the same node/edge schema.
The replay only calibrates two of four weights. AIS anomaly counts and sanctions deltas for the Feb–May 2026 window were not independently reconstructed, so only the news and price weights were grid-searched against real outcomes — the AIS and sanctions weights are still design-time estimates, not calibrated ones.
The extraction model is a free-tier LLM. Groq's free tier is used by default for cost reasons. The extraction prompt and ontology are model-agnostic — swapping in a stronger model only requires changing the client in scripts/extract.py.
