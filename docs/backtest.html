Replay scope and limitations

This replays a curated timeline of the actual Feb–May 2026 Strait of Hormuz disruption through the same compute_corridor_score() function the live dashboard uses — day by day, using only information that would have been available up to that date. It is not a live historical data pull; the event timeline was reconstructed from public reporting (Congressional Research Service, Kiel Institute, LNRG Technology, Cashu Research) for offline testing, since this environment has no way to re-query GDELT for a past date range.

What this replay does not cover: AIS anomaly counts and sanctions deltas for this specific historical window were not independently reconstructed, so only the news and price weights were calibrated this way — the AIS and sanctions weights keep their design-time defaults. That's a real gap, not a hidden one.

How calibration works

The news and price weights shown above were grid-searched to maximize how early the score crossed a high-risk threshold relative to when Brent actually crossed $100/bbl — that search is what scripts/backtest.py does, and its result is written to data/calibrated_weights.json, which the live scorer picks up automatically.
